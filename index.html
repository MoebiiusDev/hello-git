git

init

Se crea el repositorio desde 0, en teoría es el primer comando que se debería de hacer.

git

branch -m main

Renombra la rama principal, ya que esta se llamaba master pero por cosas de la vida ahora tiene otro nombre.

git

status

Muestra el estado actual de la rama, si es que se han subido archivos o no se han realizado commits.

git

add .

Añade todos los archivos a la rama, cabe recalcar que esto puede hacerse por separado.

git

commit -m ""

Nos permite agregar una copia de nuestro archivo que estaba en el area local.

git

log --graph

Nos muestra en consola en forma de rama todos los cambios que se han realizado.

git

checkout

Nos permite viajar en todos los commits que ya hayamos realizados, podrías básicamente volver al inicio del todo., se pueden ver los commit usando el "git log".

git

diff

Nos permite ver los cambios de manera mas gráfica antes de hacer commit

git

tag

En vez de identificar un commit por letra y números, con el tag podemos asignarle un nombre en concreto. (git tag nombre_cualquiera)

git

branch

Crea una nueva rama. (git branch nombre)

git

branch -d

Borra la rama.

git

switch

Accedes a las nueva ramas o a la que elijas (git switch nombre)

git

merge

Combinas los campos de las ramas. (Se va a la otra rama creada, por ejemplo "login", y haz el siguiente comando: git merge main). Traerás todos los datos de la rama main a la rama login.

git

stash

Es como generar un commit pero temporal. (git stash list es para ver todos los stash temporales)

git

pull

Se descarga el historial y los cambios, para descargar cambios que se hayan hecho en el propio GitHub se debe de poner los siguentes comandos, no preguntes, solo gozalo: git config pull.rebase false | git pull origin main

git

fetch

Se descarga el historial de los cambios

git

push

Profe explica como el pico, push nos permite subir los cambios que teníamos en el area local al GitHub

Para resumir todo esto, dejare las lineas de comandos que debes utilizar para crear tanto el repositorio, subir los archivos al area local, hacer commits y finalmente push:

git

init

git

branch -m main

git

add .

git

commit -m "Comentario o actualización"

CONEXIÓN CON GITHUB

git

remote add origin https://github.com/Nombre_de_Usuario_GitHub/Nombre_de_repositorio.git

git

push -u origin main

En el caso de conectar el repositorio con Github, al momento de crear un nuevo repositorio te sale en pantalla los mismos comandos y mas simples, para no estar escribiéndolos manualmente.
